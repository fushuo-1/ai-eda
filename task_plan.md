# 任务计划：修改扩展插件重连机制

## 任务描述
修改 `packages/pro-api-sdk/src/websocket-client.ts` 中的重连机制：
- **第一阶段**：快速重连3次（立即触发，1秒超时，总耗时≤3秒）
- **第二阶段**：指数退避3次（间隔 1秒→2秒→4秒，5秒超时）
- 全部失败后弹窗提示用户手动重连
- 启用心跳检测（15秒间隔，10秒超时，3次失败触发重连）

## 实现总结

### 断连检测方式
| 方式 | 位置 | 说明 |
|------|------|------|
| 心跳超时 | `startHeartbeat()` L411-415 | 连续3次未收到pong，触发重连 |
| 心跳发送失败 | `startHeartbeat()` L430-444 | send()异常，触发重连 |
| 消息发送失败 | `send()` L229-244 | send()异常，触发重连 |
| 错误发送失败 | `sendError()` L261-276 | send()异常，触发重连 |

### 心跳参数
| 参数 | 值 |
|------|-----|
| 发送间隔 | 15秒 |
| 响应超时 | 10秒 |
| 失败阈值 | 3次 |

### 重连流程
```
自动断开 → handleConnectionLost()
    ↓
第一阶段: quickReconnect() - 立即重连3次（每次1秒超时）
    ↓ 全部失败
第二阶段: attemptReconnect() - 退避重连3次 (1s→2s→4s，每次5秒超时)
    ↓ 全部失败
notifyReconnectFailed() - 弹窗提示
```

### 超时时间对比
| 阶段 | 单次超时 | 总耗时 |
|------|----------|--------|
| 快速重连 | 1秒 | ≤3秒 |
| 指数退避 | 5秒 | ≤20秒（含等待）|

### 重连时序
```
自动断开
    │
    ├─ 快速重连第1次（1秒超时）← 0秒
    │       ↓ 失败
    ├─ 快速重连第2次（1秒超时）← ~1秒
    │       ↓ 失败
    ├─ 快速重连第3次（1秒超时）← ~2秒
    │       ↓ 全部失败
    │
    ├─ 等待1秒
    ├─ 退避重连第1次（5秒超时）← ~4秒
    │       ↓ 失败
    ├─ 等待2秒
    ├─ 退避重连第2次（5秒超时）← ~11秒
    │       ↓ 失败
    ├─ 等待4秒
    ├─ 退避重连第3次（5秒超时）← ~20秒
    │       ↓ 全部失败
    │
    └─ 弹窗提示用户手动重连
```
