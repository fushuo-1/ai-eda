# 发现记录

## 当前重连机制分析

### 现有实现位置
`packages/pro-api-sdk/src/websocket-client.ts`

### 现有重连逻辑
```
attemptReconnect() 方法 (L467-492)
- 指数退避: 2^0*1000 = 1秒, 2^1*1000 = 2秒, 2^2*1000 = 4秒
- 最多重连3次
- 手动断开不触发重连
```

### 触发重连的场景
1. 心跳超时 (L396-400) - 连续3次未收到pong
2. 发送失败 (L223-228) - send() 异常
3. 心跳发送失败 (L420-429) - ping发送异常

### 待确认问题
1. **时间矛盾**: 用户要求"3秒内重连三次"，但间隔1s+2s+4s=7秒
2. **立即触发**: "立即触发连接"是否意味着第一次重连不等待？

## 需求理解
根据用户描述，新的重连时序应该是：
- 自动断开 → 立即第1次重连（不等待）
- 失败 → 等待1秒 → 第2次重连
- 失败 → 等待2秒 → 第3次重连
- 失败 → 等待4秒 → 弹窗提示

这样总等待时间是 0 + 1 + 2 = 3秒（第3次重连在3秒时开始）
